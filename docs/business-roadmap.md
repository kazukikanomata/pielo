# pielo 事業化ロードマップ — 案A「世界観キット」

決定日: 2026-07-08 / 方針: 事業化(副収入ではなく事業として育てる)

## プロダクト定義

**商品**: デザイン様式ごとの「世界観キット」。AIツール(v0 / Lovable / Cursor / Claude Code)や手組みの開発で、
「その様式の顔」を一発で再現するための素材一式。

1キットの中身(プロトタイプで形式を確定):

| ファイル | 内容 | 買い手にとっての価値 |
|---|---|---|
| `ai-style-prompt.md` | AIツールに貼るスタイルプロンプト(日英) | **主役。** AI出力の没個性を一発で直す |
| `theme.css` | CSS変数 + Tailwind v4 `@theme` | コピペで色・角丸・影の文法が入る |
| `tokens.json` | デザイントークン(構造化データ) | Figma変数・他ツールへの取り込み |
| `fonts-and-type.md` | フォントペア(日/英)と組み方の数値 | フォント選びの迷いを消す |
| `README.md` | 使い方 + 「なぜそう見えるか」+ 見分けチェックリスト | pieloの差別化そのもの。根拠つきの様式 |

**差別化**: 無料テーマツール(tweakcn等)は「色をいじる道具」。pieloは「歴史的根拠つきの様式39種という**編集済みの答え**」。
日本語圏ではほぼ空白地帯。

## リスクと対策

| リスク | 対策 |
|---|---|
| 無料ツールで十分と思われる | 「答え(様式)を売る」訴求に徹する。プロンプト実演のbefore/afterで差を見せる |
| SNSが伸びない | 投稿はデータから量産できるため、切り口を週次で替えてA/Bする |
| 単品が売れてもバンドルが売れない | 当たり様式のカテゴリ(例: レトロ系5種)の小バンドルを中間に挟む |
| 制作が自分の時間に依存 | キット生成をmovements.jsx+モックコードからスクリプト化する |
